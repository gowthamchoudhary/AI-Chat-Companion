=================================================
WHAT THIS APP DOES
=================================================
A personal AI news companion that a user names themselves. 
The companion:
1. Activates with the user's chosen name and voice
2. Answers any news/world event question with LIVE web data via Firecrawl Search
3. Has 3 modes the user can switch between mid-conversation:
   - CHAT MODE: Talk to your companion conversationally. Ask anything. Get short, clear answers from live web search.
   - DEBATE ME MODE: Companion takes the opposing view on the current topic and debates the user voice-to-voice using live facts
   - WATCH DEBATE MODE: Two AI agents (The Analyst + The Advocate) debate the topic live with different voices. User can interrupt anytime.

--- lib/firecrawl.ts ---
import FirecrawlApp from '@mendable/firecrawl-js'

Initialize client with FIRECRAWL_API_KEY.

Export async function searchWeb(query: string): Promise<string>
- Calls fc.search(query, { limit: 5 })
- Maps results to: "${title}: ${description}" joined by newlines
- Returns the combined string
- Wraps in try/catch, returns "No results found" on error

--- lib/elevenlabs.ts ---
Import ElevenLabsClient from @elevenlabs/elevenlabs-js

Export helper functions:
- getAgentId(mode: 'companion' | 'analyst' | 'advocate'): string
  Returns the correct env var agent ID based on mode

- createSignedUrl(agentId: string): Promise<string>
  Calls ElevenLabs API to get a signed WebSocket URL for the session
  Uses ELEVENLABS_API_KEY

--- app/api/search/route.ts ---
POST endpoint.
Body: { query: string }
- Calls searchWeb(query) from lib/firecrawl.ts
- Returns: { result: string }
- This is the webhook URL you register in ElevenLabs dashboard for all 3 agents

--- app/api/agent-token/route.ts ---
POST endpoint.
Body: { agentId: string }
- Calls ElevenLabs API to get signed session token
- Returns: { signedUrl: string }
- This keeps the API key server-side, never exposed to client

--- app/api/verdict/route.ts ---
POST endpoint.
Body: { transcript: Message[] }
- Takes the debate transcript
- Calls OpenAI GPT-4o-mini with a simple prompt:
  "Summarise this debate. Give 2 bullet points for The Analyst's strongest arguments and 2 for The Advocate's. Then one line verdict: who made the stronger case and why. Be concise."
- Returns: { verdict: string }

--- components/ModeBar.tsx ---
Three mode buttons:
- 💬 Chat (default)
- ⚔️ Debate Me  
- 👁️ Watch Debate

Active mode highlighted. On mode switch:
- Update Zustand mode
- If switching to 'debate-me': disconnect companion session, connect analyst session, inject current topic context
- If switching to 'watch-debate': connect BOTH analyst + advocate sessions, start turn-taking loop
- If switching back to 'chat': disconnect debate sessions, reconnect companion

--- components/DebateArena.tsx ---
Only visible when mode is 'watch-debate'.
Two-panel layout side by side:
- Left panel: The Analyst (blue theme) — name, speaking indicator, their transcript lines
- Right panel: The Advocate (coral/red theme) — name, speaking indicator, their transcript lines
- Center: VS badge
- Below both: interrupt button "Jump In" → opens user mic

Turn-taking logic:
- Simple state: whose turn it is
- When Analyst finishes speaking → trigger Advocate to respond
- When Advocate finishes → trigger Analyst
- Repeat until user interrupts or switches mode
- After 4 full exchanges → auto-call /api/verdict → show VerdictCard

IMPORTANT NOTES FOR THE AGENT
=================================================
- Keep ALL API keys server-side only (in API routes). Never expose to client.
- The ElevenLabs WebSocket connection happens CLIENT-SIDE using the signed URL from /api/agent-token
- Firecrawl Search is called SERVER-SIDE only (via the webhook from ElevenLabs → your API route)
- For the turn-taking in watch-debate mode: use a simple boolean state + useEffect to trigger the next agent when the current one finishes. Do NOT overcomplicate this.
- The companion's voiceId is dynamic — it should be passed when creating the agent session, not hardcoded
- Handle all loading and error states — show the user something meaningful if an API call fails
