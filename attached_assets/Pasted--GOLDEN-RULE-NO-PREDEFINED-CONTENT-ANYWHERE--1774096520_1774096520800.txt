HOW IT WORKS — THE EXACT MECHANISM
=================================================

ElevenLabs has a feature called CLIENT TOOLS.
A client tool is a tool the agent calls that instead 
of hitting a server — it sends an event directly 
to your frontend JavaScript.

Flow:
1. User asks "What's new with Claude?"
2. Agent calls search_web webhook → Firecrawl returns results
3. Agent ALSO calls show_cards client tool → sends card data to frontend
4. Frontend receives the card data → renders news cards
5. Cards slide in WHILE the agent is still speaking the answer
6. User says "show me the post" → agent calls show_cards again with expanded view

STEP 1 — UPDATE /api/search/route.ts
=================================================

The search route currently returns just { result: string }
Update it to return BOTH the spoken text AND the structured card data:

export async function POST(req) {
  const { query } = await req.json()
  
  const results = await fc.search(query, { limit: 5 })
  
  // Text for agent to speak — short summaries only
  const spokenText = results.data
    .slice(0, 3)
    .map(r => `${r.title}: ${r.description}`)
    .join('\n')
  
  // Structured data for visual cards
  const cards = results.data.slice(0, 5).map(r => ({
    title: r.title || '',
    description: r.description || '',
    url: r.url || '',
    image: r.metadata?.ogImage || r.metadata?.image || null,
    source: new URL(r.url).hostname.replace('www.', '') || '',
    publishedDate: r.metadata?.publishedDate || null,
  }))
  
  return Response.json({ 
    result: spokenText,  // what agent speaks
    cards: cards         // what frontend renders
  })
}

STEP 4 — ADD CLIENT TOOL IN ELEVENLABS DASHBOARD
=================================================

In ElevenLabs dashboard, for EACH of your 3 agents,
add a CLIENT TOOL (not a webhook — a client tool):

Tool name: show_cards
Description: "Display relevant news cards on screen when answering a news question. Call this EVERY TIME you use search results to answer. Pass the search results as cards."
Parameters:
  - cards (array) — array of objects with: title, description, url, image, source, publishedDate
  - query (string) — the user's original question

This tool does NOT hit a server URL.
It sends an event to the frontend WebSocket connection.
The frontend listens for this event and renders the cards.

=================================================
STEP 5 — LISTEN FOR CLIENT TOOL IN FRONTEND
=================================================

In your ElevenLabs WebSocket connection setup,
add an event listener for client tool calls.

Using the ElevenLabs JS SDK:

conversation.on('client_tool_call', (toolCall) => {
  if (toolCall.tool_name === 'show_cards') {
    const { cards, query } = toolCall.parameters
    // Update Zustand store with new cards
    useAppStore.getState().setNewsCards(cards)
    useAppStore.getState().setCurrentQuery(query)
    // Tool must return a result to the agent
    return { result: 'Cards displayed successfully' }
  }
})

Add to Zustand store:
- newsCards: CardData[]
- currentQuery: string
- setNewsCards(cards: CardData[]): void
- setCurrentQuery(query: string): void

IMPORTANT TECHNICAL NOTES
=================================================

- The show_cards client tool is the critical integration.
  If ElevenLabs SDK doesn't support client_tool_call event,
  use this alternative: store cards in /api/search response,
  have frontend poll a temporary endpoint, or use a 
  server-sent event (SSE) to push cards to frontend when 
  Firecrawl returns results.

- For images: many news sites block og:image hotlinking.
  If image fails to load → show a gradient placeholder with
  the source name's first letter as a large centered character.

- date-fns for "2h ago" formatting:
  npm install date-fns
  import { formatDistanceToNow } from 'date-fns'

- html2canvas for share image:
  npm install html2canvas
  
- For the scrape route — only scrape when user explicitly 
  asks to see more. Don't auto-scrape. Firecrawl credits 
  are limited.

- Mobile first. The demo video will likely be filmed on 
  a phone screen. Every component must look good at 390px width.
